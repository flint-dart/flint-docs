Sessions & Cookies

Flint includes session helpers built on cookies. Use req.startSession(), req.session, and req.destroySession() to manage login state.

Start a Session

app.post('/login', (req, res) async {
  final user = {'id': 1, 'email': 'user@example.com'};
  await req.startSession(user, ttl: Duration(hours: 8));
  return res.json({'ok': true});
});

Cookies

Sessions are stored server-side and linked by a cookie named FLINTSESSID. The cookie is set automatically when you call req.startSession(). 

final sessionId = req.cookies['FLINTSESSID'];
final isAuthed = sessionId != null;

You can also set your own cookies directly on the response: 

app.get('/set-cookie', (req, res) async {
  res
    .setCookie('theme', 'dark', maxAge: 3600, httpOnly: true, sameSite: 'Lax')
    .setCookie('lang', 'en', path: '/');
  return res.json({'ok': true});
});

app.get('/clear-cookie', (req, res) async {
  res.clearCookie('theme');
  return res.json({'ok': true});
});

Read Session Data

app.get('/me', (req, res) async {
  final session = await req.session;
  if (session == null) return res.status(401).json({'error': 'Unauthorized'});
  return res.json({'user': session});
});

Update or Destroy

await req.updateSession({'role': 'admin'});
await req.destroySession();

Sessions are powered by CookieSessionMiddleware, which is always registered by default.

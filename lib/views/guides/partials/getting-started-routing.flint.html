Routing

Define routes in lib/main.dart or a route group.

import 'package:flint_dart/flint_dart.dart';

void main() {
  final app = Flint(
    withDefaultMiddleware: true,
    enableSwaggerDocs: true,
    autoConnectDb: false,
  );

  app.get('/', (req, res) async {
    return res.json({'status': 'ok'});
  });

  app.get('/users/:id', (req, res) async {
    final id = req.params['id'];
    return res.json({'id': id});
  });

  app.post('/users', (req, res) async {
    final data = await req.json();
    return res.json({'created': true, 'data': data});
  });

  app.listen(port: 3000);
}

Attach middleware per route:

app.post('/profile', handler)
  .use(AuthMiddleware());

Route Params

Use :param segments. Read them with req.param(). The raw map is req.params.

app.get('/users/:id', (req, res) async {
  final id = req.param('id');
  return res.json({'id': id});
});

Query Params

Read query string values from req.queryParam() or req.query (e.g. ?page=2&limit=10).

app.get('/users', (req, res) async {
  final page = req.queryParam('page') ?? '1';
  final limit = req.queryParam('limit') ?? '10';
  return res.json({'page': page, 'limit': limit});
});

Request & Response

Common helpers you’ll use in most handlers.

app.post('/users', (req, res) async {
  final body = await req.json(); // parse JSON body
  final token = req.bearerToken; // read Authorization token
  final ip = req.ipAddress; // client IP

  return res
      .status(201)
      .json({'created': true, 'data': body, 'ip': ip, 'token': token});
});

Response Methods

Use the right response method based on what you’re returning.

app.get('/text', (req, res) async {
  return res.send('Plain text');
});

app.get('/json', (req, res) async {
  return res.json({'status': 'ok'});
});

app.get('/auto', (req, res) async {
  return res.respond({'auto': 'json'}); // infers JSON/HTML/text
});

app.get('/page', (req, res) async {
  return res.view('home', data: {'title': 'Flint Docs'});
});

send() is for plain text or custom content types. json() sets JSON headers and encodes safely. respond() auto-detects the best type. view() renders a template.

Request Body

Choose the right body reader based on the incoming content type.

app.post('/raw', (req, res) async {
  final text = await req.body(); // raw string body
  return res.send(text);
});

app.post('/json', (req, res) async {
  final data = await req.json(); // Map<String, dynamic>
  return res.json({'received': data});
});

app.post('/form', (req, res) async {
  final fields = await req.form(); // Map<String, String>
  return res.json({'fields': fields});
});

body() returns a raw string. json() parses JSON into a map. form() reads application/x-www-form-urlencoded or multipart/form-data.

File Uploads & Storage

Uploads come from multipart/form-data. You can read files, check if they exist, or store them on disk.

  • req.file('avatar') — get a single uploaded file.
  • req.files('photos') — get multiple files for the same field.
  • req.hasFile('avatar') / req.hasFiles('photos') — check if files were sent.
  • req.storeFile(...) / req.storeFiles(...) — save to disk and return path(s).
app.post('/avatar', (req, res) async {
  // Access the uploaded file
  final upload = await req.file('avatar');
  if (upload == null) {
    return res.status(400).json({'error': 'No file uploaded'});
  }

  // Save with a custom name
  final path = await req.storeFile(
    'avatar',
    directory: 'public/uploads',
    filename: 'user-1.png',
  );

  return res.json({'saved_to': path});
});
app.post('/gallery', (req, res) async {
  if (!await req.hasFiles('photos')) {
    return res.status(400).json({'error': 'No photos uploaded'});
  }

  final paths = await req.storeFiles('photos', directory: 'public/uploads');
  return res.json({'saved_to': paths});
});

Tip: you can also save manually using upload.saveTo(path) if you want full control.
